game-of-life / Conway's Game of Life

JavaFX 实现的康威生命游戏 —— 项目功能介绍 & 运行指南。

＊ 这是什么

Java 11 Maven JavaFX 17 TestNG givedrug.math

一个能在屏幕上跑 康威生命游戏（Conway's Game of Life） 的 JavaFX Demo。 支持从 conwaylife.com 标准的 .cells plaintext 格式加载初始图样，启动后用 AnimationTimer 持续演进， 在单个 Canvas 上渲染。

内置两个图样：gosperglidergun（最经典的 Gosper 滑翔机枪） 和 breeder1（Bill Gosper 1970s 发现的二次增长图样）。

＊ 核心规则

每个格子有"活/死"两态。每一步根据当前格子的 8 邻居（上下左右 + 4 个对角）计算下一帧：

  1. 活 → 死（孤独）：活格子的活邻居数 < 2
  2. 活 → 死（拥挤）：活格子的活邻居数 > 3
  3. 活 → 活（存活）：活格子的活邻居数 = 2 或 3
  4. 死 → 活（繁殖）：死格子的活邻居数 = 3

＊ 功能特性

渲染

  • 单 Canvas + GraphicsContext.fillRect，支持百万级格子稳定渲染
  • 增量重绘：每帧只重绘"状态翻转"的格子，CPU 开销与活跃区域大小相关而非画布总面积

演化算法

  • 稀疏活细胞集合：用 Set<Long>（坐标编码成 long）只保存活细胞
  • 每代复杂度约 O(A)，A 是活细胞数 —— 比稠密遍历 O(W·H) 在稀疏图样下快几个数量级
  • 边界：硬边界（出界视为死格子）

图样加载

  • 支持 conwaylife.com 标准的 .cells plaintext 格式
  • 自动测尺寸：解析时统计行数和最大行长，调用方不再手填 boundx/boundy
  • 注释行（! 开头）和空行自动跳过

交互控制

  • 键盘控制：暂停/单步/调速/重置/清屏
  • 鼠标点击：切换单个格子的状态（暂停时可手画图样）
  • 窗口标题实时显示：图样名 / 代数 / 活细胞数 / 演进速率 / 暂停状态

＊ 怎么跑

前置条件

  • JDK 11+（推荐 JDK 17 / 21；JavaFX 17 已作为 Maven 依赖引入，无需单独装 JavaFX）
  • Maven 3.6+

本机 macOS 上的常用 JDK 切换：

export JAVA_HOME=$(/usr/libexec/java_home -v 21)

运行（默认 breeder1）

cd /Users/mi/game-of-life
mvn javafx:run

启动后会弹出一个 1500×700 的窗口，可以看到 Breeder 1 一边走一边喷出滑翔机枪， 活细胞数随时间 t 增长为 O(t²)。

切换图样：Gosper 滑翔机枪

mvn javafx:run -Djavafx.args="gun"

支持的参数（大小写不敏感）：

参数对应图样画布
gun / gosper / gosper-glider-gunGosper 滑翔机枪200×100，cell=6px
breeder / breeder1（默认）Breeder 11500×700，cell=1px

注 新加图样：把 .cells 文件丢到 src/main/resources/， 在 PatternPreset.java 加一条枚举即可，不用再手填尺寸。

跑测试

mvn test

包含 3 个 TestNG 用例：图样加载校验、稳定块（block）、振荡器（blinker）。

只编译不跑

mvn compile        # 编译主代码
mvn package        # 打成 jar 到 target/

＊ 键盘 & 鼠标

按键 / 操作作用
Space暂停 / 继续
S单步推进一代（仅在暂停时）
↑加速（每代间隔减半，下限 8ms）
↓减速（每代间隔加倍，上限 2s）
R重置当前预设到初始图样
C清空整个画布
鼠标点击切换被点击格子的状态（活↔死）

小贴士 先 C 清屏 → 鼠标点击画自己的图样 → Space 启动，可以当个简易沙盒。

    └── test/java/givedrug/math/gameoflife/
        └── LoadPatternFileTest.java  # 加载器 + 演化算法的 TestNG 用例

＊ 架构概览

类职责
GameOfLife JavaFX Application 入口；持有 Canvas，处理键盘/鼠标输入；AnimationTimer 按 stepIntervalNanos 驱动演进；增量重绘翻转的格子。
GameEngine 纯算法层，与 JavaFX 解耦。step() 演进一代并返回"状态翻转的格子集合"，供调用方做增量重绘。可被单元测试直接覆盖。
LoadPatternFile 从 classpath 读 .cells，扫描注释行，统计宽高，输出 Pattern。
Pattern 不可变值对象：宽 / 高 / 活细胞 long 编码数组。坐标编码 (x << 32) | y。
PatternPreset 内置图样预设枚举：文件名 + 画布尺寸 + cell 边长 + 偏移；附 fromArg() 解析命令行。

＊ 加自己的图样

  1. 从 conwaylife.com wiki 找一个图样，下载 .cells 文件。
  2. 放到 src/main/resources/ 下，比如 acorn.cells。
  3. 在 PatternPreset.java 加一条枚举： 
    ACORN(
        "acorn.cells",
        /* canvasWidth */ 400,
        /* canvasHeight */ 300,
        /* sideLength */  3,
        /* shiftX */      150,
        /* shiftY */      120),
  4. 在 fromArg() 的 switch 加个别名（可选）。
  5. 运行：mvn javafx:run -Djavafx.args="ACORN"

注 画布尺寸要给"演化扩散"留余量。Acorn 跑 5206 代才稳定下来，需要至少 200×200 的活动空间。

＊ 常见问题

报 Error: JavaFX runtime components are missing

用 mvn javafx:run 跑，不要用 java -jar。或者把 JavaFX module 显式 --add-modules。

JDK 8 能不能跑

不行。新 pom 已升到 Java 11 + JavaFX 17。要跑 JDK 8 版本可以 git checkout 35ff2ed 看老代码。

Breeder1 跑久了卡

Breeder 是二次增长图样，活细胞数随时间平方增长，迟早会撑爆 1500×700 画布。撞到边界后会出现"边缘异常"，这是硬边界设计的预期行为。

窗口太大屏幕装不下

调小 PatternPreset.BREEDER1 的 canvasWidth/canvasHeight，或者加大 sideLength 让 cell 变大但格子数变少。

＊ 一句话总结

一个 能跑得动的康威生命游戏：单 Canvas 渲染 + 稀疏 Set 演化， 丢一个 .cells 文件 + 加一条枚举，就能看任意图样在屏幕上活过来。 mvn javafx:run 启动，Space 暂停，鼠标画几笔，享受。
